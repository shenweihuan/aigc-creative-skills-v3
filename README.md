## 快速开始

### 方式1：直接说需求（推荐）

直接用自然语言描述你想做什么，MetaSkill 总控会自动路由：

```
"用莫奈的风格画一张秋天的湖边小屋"
→ 自动路由：AI绘画大师 + 莫奈印象派技能

"帮我用海明威的笔法写一段加班的感受"
→ 自动路由：AI写作大师 + 海明威极简技能

"做一个赛博朋克风格的15秒城市短片"
→ 自动路由：AI视频大师 + （推荐风格匹配的大师）

"我不知道用什么风格，帮我推荐"
→ 运行风格诊断流程 → 推荐最适合你的大师组合
```

### 方式2：先学再做

如果你想系统学习：

```
Step 1: 读 shared/00-creation-philosophy.md（20分钟，建立世界观）
Step 2: 选一个你感兴趣的领域（视觉/写作/影视/设计）
Step 3: 在该领域选一个艺术大师作为入门（建议从莫奈/海明威开始）
Step 4: 完成该大师技能的全部训练和实战
Step 5: 用对应的 AI 大师技能执行你的第一个完整项目
```

### 方式3：只学工艺

如果你只想快速掌握 AI 工具的使用方法：

```
→ 直接加载对应领域的 AI 大师技能
→ 进入教学模式（Mode A）
→ 按平台指南选择工具
→ 用工作流模式（Mode C）规划你的第一个项目
```

## 核心功能一览

### 智能路由

不需要记住30+个技能的名字。自然语言描述需求，系统自动：
1. 识别创作类型（图/文/视频/设计）
2. 匹配最合适的艺术大师风格
3. 加载对应的 AI 工具专家
4. 组合两者开始工作

### 四种工作模式

每个 AI 大师都支持四种模式，按需切换：

| 模式 | 你说 | 系统做 |
|------|------|--------|
| **教学** | "教我用AI画画" | 平台选择 + 技法讲解 + 速查卡 |
| **评价** | "评价这张图" | 多维评分 + 缺陷诊断 + 修改方案 |
| **工作流** | "帮我制定计划" | 项目模板 + 分阶段规划 |
| **自动迭代** | "还不够好再来" | 生成→评价→修改 循环（最多5轮） |

### 艺术大师深度体验

每个艺术大师技能包含：

| 模块 | 内容 | 价值 |
|------|------|------|
| 大师档案 | 为什么是这位大师？核心美学命题 | 不只是"有名"，而是解决了一个具体的审美问题 |
| 感知训练 | 3-5个具体练习 | 训练你的眼睛/耳朵/心，而不只是教技术 |
| 提示词公式 | 该大师风格的专属 prompt 模板 | 不是通用模板，而是有灵魂的公式 |
| 评价标准 | 5级自检清单 | 学会用大师的标准判断作品质量 |
| 跨域迁移 | 这个品味还能用在哪里 | 从绘画到写作到设计的迁移路径 |

## 与 v1 / v2 的关系

| 版本 | 规模 | 架构 | 核心理念 |
|------|------|------|---------|
| **v1** | 12,182行 / 8模块 | 平铺领域 | 教你用AI工具（方法堆砌） |
| **v2** | ~3,500行 / 5层 | 过程分层 | 教你怎么思考（过程导向） |
| **v3** | 目标 每个800-1000行 × N | **MetaSkill + 原子技能** | **教你怎么像大师一样感知和创造** |

v3 不替代 v1/v2。v3 的 shared/ 模块继承自 v2 的 L0(哲学) 和 L4(循环)。

## 设计原则

1. **品味 > 技能** — 每个技能的核心是审美训练，不是操作手册
2. **深 > 广** — 每个大师技能 800-1000 行深内容，不追求覆盖面
3. **一次一个** — 质量优先于速度，每个技能都是精品才发布
4. **可组合** — 基于 OpenSquilla MetaSkill 规范，技能间可动态组合
5. **闭环** — 创作→评价→迭代→沉淀，完整的数据流
