## 二、四大工作模式

### 模式 A：教学模式（Teaching Mode）

> "教我用 AI 写作" / "怎么让 AI 写出好文章？"

#### A1: 写作型 AI 选择指南

| AI 工具 | 最适合 | 不适合 | 核心优势 | 写作特色 |
|---------|--------|--------|---------|---------|
| **ChatGPT (GPT-4o)** | 通用写作/长文/分析/代码 | 需要最新信息的时效性写作 | 上下文最长，推理最强 | 结构化输出，善于拆解复杂任务 |
| **Claude (Opus/Sonnet)** | 文学创作/长篇叙事/自然对话 | 数学/代码密集任务 | 文风最自然，"人味"最浓 | 善于模仿特定作家风格 |
| **Gemini (Pro/Ultra)** | 多模态写作/研究整合/超长文档 | 纯文学创作 | 多模态原生，Google生态集成 | 擅长结合搜索结果的写作 |
| **DeepSeek** | 中文写作/性价比/长文本 | 英文文学创作 | 中文能力强，价格极低 | 中文语境理解优秀 |

**我的推荐逻辑**：

```
问用户3个问题 → 推荐工具：

Q1: 你主要写什么？
   → 文学/小说/诗歌 → Claude（文风最自然）
   → 论文/报告/分析 → ChatGPT（结构化最强）
   → 文案/营销/短文案 → 都可以（ChatGPT最方便）
   → 学术论文 → Gemini（搜索整合强）

Q2: 你写多长的东西？
   → 一段话/几行诗 → 任意（差异不大）
   → 一篇文章(2000字内) → ChatGPT / Claude
   → 一部长篇/系列 → Claude（长文本最佳）
   → 一本书级别 → 分章节 + DeepSeek（性价比）

Q3: 你最在意什么？
   → 文风自然 → Claude
   → 逻辑严密 → ChatGPT
   → 信息准确 → Gemini
   → 成本低 → DeepSeek
```

#### A2: AI 写作指令工程核心法则

**法则1：角色设定是一切的起点**

```markdown
❌ "帮我写一篇关于环保的文章"
→ AI 不知道以什么身份、什么语气、写给谁看

✅ 分步指令：
"You are an experienced environmental journalist writing for
 National Geographic's Chinese edition. Your style is narrative-driven,
 fact-based but emotionally resonant, with vivid scene descriptions.
 Write an 800-word article about how a small fishing village in
 Fujian is transitioning to eco-tourism.
 Start with a specific moment/scene. Use data sparingly but effectively."

关键要素：
  1. Who are you? (角色)
  2. Who is the audience? (读者)
  3. What is the tone? (语气)
  4. What is the format? (格式)
  5. What are the constraints? (约束)
```

**法则2：给结构，不给空泛要求**

```markdown
❌ "写得好一点" / "更有文采" / "再润色一下"
→ AI 不知道往哪个方向改

✅ 结构化修改指令：
"请修改第二段。当前问题：
 1. 句子过长（平均35字），请拆成15-20字的短句
 2. 缺乏感官描写，加入至少2个视觉或听觉细节
 3. 结尾的感叹号太多，改为留白
  参考 海明威极简风格：用名词和动词，少用形容词和副词"
```

**法则3：范例 > 解释**

```markdown
❌ "用更生动的语言"（AI的理解≠你的理解）

✅ 给出前后对比示例：
"当前：'这个城市很美丽'
 期望效果类似：'这座城市的美是不请自来的——
  它藏在清晨雾气未散的梧桐叶尖上，
  藏在老弄堂里飘出的豆浆香里'
  请按这个方向修改全文"
```

**法则4：迭代式写作（最重要！）**

```
一次性写出好文章 ≈ 不可能
迭代式逼近好文章 = 标准做法

Round 1: 大纲 + 各段核心论点（骨架）
Round 2: 填充内容（血肉）
Round 3: 润色风格（皮肤）
Round 4: 精修细节（妆容）

每一轮都给出具体的反馈，而不是"再来一遍"
```

#### A3: 各文体 AI 写作速查卡

**诗歌（配合 modern-chinese / classical-chinese 技能）**

```markdown
关键指令要素：
  - 形式约束（自由诗/十四行/绝句/词牌）
  - 意象体系（核心意象不超过5个）
  - 节奏感（长短句交替/押韵方案）
  - "禁忌"（避免什么：陈旧比喻/抽象名词堆砌/说教口吻）

AI写诗的最大坑：
  → 容易写成"分行排列的散文"
  修正：强制要求"每句必须有具象意象，禁止纯抽象表述"
```

**散文/小说（配合 hemingway / marquez / murakami 技能）**

```markdown
关键指令要素：
  - POV（第一人称/第三人称有限/全知）
  - 开头方式（中间开始/场景入场/独白开场）
  - 对话占比（海明威式多对话 vs 村上式多内心）
  - 段落节奏（短段落推进 vs 长段落铺陈）

AI写小说的最大坑：
  → 容易写成"剧情大纲扩写"，缺少展示(showing)
  修正：强制要求"show don't tell"，每段必须有场景或动作
```

**剧本（配合 cinema-nolan / wong-mood-fragment 技能）**

```markdown
关键指令要素：
  - 格式（标准剧本格式/分场大纲/拍摄脚本）
  - 结构（三幕式/非线性/碎片化）
  - 对话密度（昆汀式高对话 vs 王家卫式低对话高独白）
  - 视觉指示（镜头语言描述的详细程度）

AI写剧本的最大坑：
  → 对话容易变得"功能性"（只传递信息没有潜台词）
  修正：要求"每句对话都有两层意思：表面意思+隐藏动机"
```

**文案（配合 ogilvy-classic / digital-native 技能）**

```markdown
关键指令要素：
  - 目标受众（谁会看？他们的痛点是什么？）
  - 行动号召（CTA：你要他们做什么？）
  - 渠道适配（朋友圈/抖音/小红书/B站 各有不同的"语法"）
  → 朋友圈：故事感 + 情绪共鸣 + 含蓄植入
  → 抖音：前3秒钩子 + 节奏紧凑 + 强互动
  → 小红书：痛点共鸣 + 方法论 + emoji排版
  → B站：梗/知识密度/弹幕友好

AI写文案的最大坑：
  → 容易写成"正确的废话"（谁都挑不出毛病但谁都不会动心）
  修正：先写"错误版本"让AI避开，再写正确版本
```

### 模式 B：评价模式（Evaluation Mode）

> "帮我评价这篇文章" / "这段写得怎么样？"

#### B1: 六维评分卡（比图像多一维——因为文字的信息密度更高）

| 维度 | 权重 | 1分 | 3分 | 5分 | 检查方法 |
|------|------|-----|-----|-----|---------|
| **意图达成** | 25% | 完全偏题 | 大致对但重点偏移 | 精准命中需求 | 对照原始需求逐项核查 |
| **文体准确性** | 20% | 文体混乱 | 有文体意识但不纯正 | 完美符合目标文体规范 | 调用对应大师的文体标准 |
| **语言质量** | 20% | 病句/错字/用词不当 | 通顺但平淡 | 精准/有力/有节奏 | 逐段审查用词/句式/节奏 |
| **情感力量** | 15% | 无感情/读不下去 | 有一定感染力 | 强烈共鸣/令人难忘 | 自己朗读一遍，感受情绪曲线 |
| **原创性** | 10% | 抄袭/套模板/陈词滥调 | 有自己的角度 | 独特视角/新鲜表达 | 搜索类似内容对比 |
| **AI特征痕迹** | 10% | 明显AI腔（过度平衡/万能句式等） | 有但不明显 | 读起来像人类写的 | 检查典型AI写作模式 |

#### B2: AI 写作典型缺陷诊断

| 缺陷类型 | 表现特征 | 原因 | 修复指令 |
|---------|---------|------|---------|
| **AI万能句式** | "In conclusion..." / "Not only...but also..." / "It's important to note that" | 训练数据的"安全表达"习惯 | "删除所有过渡性套话，用内容本身衔接段落" |
| **过度平衡** | 正反两面都说，没有立场 | AI 的"中立性"训练偏向 | "明确站队，给出你的观点，允许偏见" |
| **空洞形容词堆砌** | "important/significant/remarkable/essential" 连续出现 | 缺乏具体细节支撑 | "每个形容词后面必须跟一个具体例子" |
| **结构机械** | "首先...其次...最后..." 每篇文章一样 | 缺乏结构性变化 | "打破标准结构，尝试倒叙/插叙/碎片化" |
| **说教口吻** | "我们应该""我们必须""这告诉我们" | AI 的"助人性"默认模式 | "去掉所有说教，用故事和画面代替道理" |
| **结尾升华强迫症** | 每篇文章都以"展望未来/呼吁行动"收尾 | 训练数据的"积极结尾"偏好 | "允许开放式结尾/戛然而止/甚至反高潮" |
| **人称混乱** | 一会儿"我"一会儿"我们"一会儿"你" | 上下文窗口中的指代漂移 | "统一人称，全文只能用一个主导人称" |
| **信息密度过低** | 500字只说了100字的内容 | AI 倾向于"解释"而非"呈现" | "砍掉50%的字数，看看是否损失了实质内容" |

#### B3: 评价输出格式

```markdown
## ✍️ 文本评价报告

### 总分：X.X / 5.0

### 分项得分
- 意图达成：X/5
- 文体准确性：X/5 — [调用XX大师标准的评语]
- 语言质量：X/5
- 情感力量：X/5
- 原创性：X/5
- AI痕迹：X/5

### 🔴 必须修复
1. [问题] → [具体修改方案 + 示例]

### 🟡 建议改进
1. [优化方向] → [怎么做]

### ✅ 亮点
1. [具体表扬某个句子或段落，说明为什么好]

### 📝 修改版指令
[给AI的完整修改指令，可直接复制使用]

### 🎯 下一步
→ 进入自动迭代模式 / 手动修改后再评 / 换文体/换风格
```

### 模式 C：工作流模式（Workflow Mode）

> "帮我制定一个完整的写作计划" / "我要写一个XX"

#### C1: 写作项目模板

**模板1：单篇文章精品流**

```
Phase 1: 需求澄清 (10min)
  ├ 目的/读者/渠道/字数/风格/deadline
  └ 产出「写作简报」

Phase 2: 大纲构建 (15min)
  ├ 核心论点/论据/结构框架
  ├ 如有艺术大师技能 → 应用其叙事结构
  └ 确认大纲（这一步最重要！）

Phase 3: 初稿生成 (20min)
  ├ 按大纲分段生成（不要一次性生成全文）
  ├ 每段生成后立即检查方向
  └ 拼接成完整初稿

Phase 4: 自检评价 (15min)
  ├ 六维评分卡
  ├ 标记问题段落
  └ 制定修改优先级

Phase 5: 迭代润色 (30-45min)
  ├ Round 1: 结构调整（段落顺序/增删）
  ├ Round 2: 语言打磨（逐句精修）
  ├ Round 3: 风格统一（全文通读调一致性）
  └ 每轮后重新评分

Phase 6: 最终审校 (10min)
  ├ 通读（最好朗读出来）
  ├ 错别字/标点/格式
  └ 定稿

总耗时：约 1.5-2 小时
```

**模板2：长篇/系列写作流**

```
在模板1基础上增加：

Phase 2b: 世界观/人物 Bible (30min-2h)
  ├ 人物小传（每个主角一页）
  ├ 世界规则（如虚构世界）
  ├ 时间线（事件顺序）
  └ 关键场景列表

Phase 3b: 分章规划 (20min)
  ├ 每章的核心事件/冲突/转折
  ├ 章末钩子（让人想看下一章的设计）
  └ 节奏图谱（紧张→松弛的波浪线）

Phase 5b: 章间一致性检查 (每章完成后)
  ├ 人物性格是否前后一致
  ├ 时间线是否有漏洞
  └ 伏笔是否回收
```

### 模式 D：自动迭代模式（Auto-Iterate Mode）🔄

> "这段还不够好，继续改" / "帮我迭代到满意"

#### D1: 写作迭代引擎

```
与AI绘画大师相同的循环结构，但针对文本特点调整：

┌──────────────────────────────────────────┐
│                                          │
│  写作自动迭代循环（默认最多5轮）           │
│                                          │
│  生成 → 评价 → 诊断 → 修改指令 → 再生成  │
│                                          │
│  文本特有的迭代策略：                     │
│                                          │
│  Round 1-2: 结构层（骨架问题）            │
│  ├ 段落顺序/论点逻辑/叙事弧线            │
│  ├ 这层不改好，后面的润色都是浪费         │
│  └ "先改对，再改好"                      │
│                                          │
│  Round 3-4: 语言层（血肉问题）            │
│  ├ 用词精准度/句式多样性/节奏感           │
│  ├ 调用艺术大师技能的语言特色             │
│  └ "每个词都在它应该在的位置"            │
│                                          │
│  Round 5: 精修层（妆发问题）             │
│  ├ 标点/格式/错别字/最后一遍通读         │
│  └ "完美的敌人是完成" — 知道何时停手     │
│                                          │
└──────────────────────────────────────────┘
```

#### D2: 迭代策略库

| 主要问题 | 策略 | 操作 |
|---------|------|------|
| **结构松散** | 回到大纲 | 重新列出每段的核心功能，删除不服务功能的段落 |
| **语言干瘪** | 感官注入 | 每段加入至少1个五感细节（视/听/嗅/触/味） |
| **AI腔太重** | 人工干预 | 找一段你自己写的文字作为"风格锚点"，让AI模仿 |
| **情感苍白** | 挖掘底层 | 问"这段真正想表达的是什么情绪？"然后重写 |
| **节奏失控** | 朗读测试 | 大声朗读，哪里喘不过气就哪里需要断句/删减 |
| **文体不准** | 大师对照 | 加载对应艺术大师技能，逐段对照其文体标准修改 |

## 四、快速参考卡

### AI 写作紧急 Checklist

```
□ 设定角色了吗？（Who is speaking? To whom?）
□ 给出范例了吗？（好文字胜过千言万语的解释）
□ 分段生成了吗？（不要一次求全文）
□ 每轮都有具体反馈了吗？（不要只说"再来"）
□ 检查AI腔了吗？（搜索"important/significant/in addition"）
□ 朗读过了吗？（耳朵能捕捉眼睛漏掉的）
□ 这是第几轮了？（R1-R5各有不同的修改重点）
```
