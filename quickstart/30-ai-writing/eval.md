# AI写作 (AI Writing) — 评测套件 (Eval Suite)

> **Karpathy's Rule:** If you can't test it, you don't know if it works.
> 本文件提供标准化测试、质量基线和 before/after 验证。

| 属性 | 值 |
|------|---|
| Skill | 30-ai-writing |

## Test Prompts（3 个梯度）

### Test 1：最小可用性（新手能直接用吗？）
**Prompt:** "帮我写一条微博推广一款新出的降噪耳机"
**Expected Output Type:** 社交媒体短文案（可直接发布或微调后发布）
**Must Include:**
- 符合微博平台的语体特征（口语化/有梗/有互动引导/控制在140字内或合理长度）
- 产品卖点提炼（不是罗列参数，而是转化为用户利益点）
- 至少一个记忆点（金句/反转/共鸣/悬念）
- 话题标签建议（增加曝光）
- 配图建议（文案+视觉的配合思路）

**Quality Bar:**
- ✅ Good: "【场景切入】地铁上突然全世界安静了...【痛点共鸣】每天通勤被噪音消耗的电量...【产品解法】XX耳机一键切换...【金句】'世界很吵，你的世界可以很安静' #降噪耳机 #通勤救星"
- ❌ Bad: "隆重介绍我们公司的新款降噪耳机！采用先进的主动降噪技术，续航长达40小时，舒适佩戴，性价比超高，快来购买吧！#新品发布 #耳机推荐"

### Test 2：核心能力（这个 skill 的独特价值在哪？）
**Prompt:** "写一篇关于'远程工作三年后，我发现效率是个伪命题'的深度文章"
**Expected Output Type:** 有观点深度的长文（1500-2500字级别）
**Must Include:**
- 明确的核心论点（不是"各有优劣"的和稀泥）
- 论证结构（现象→拆解→证据→反驳→结论，有逻辑链条）
- 具体的个人体验/案例/数据（不是空洞的说理）
- 文体意识（是随笔/评论/还是叙事？语气和节奏匹配文体）
- 至少一个"反直觉"的观点或洞察（让读者觉得"没想到"）

**Quality Bar:**
- ✅ Good: 开篇用个人故事钩住读者 → 定义"效率伪命题"的具体含义 → 用3个维度展开（ measurable output / deep work / serendipity）→ 承认反面论点 → 结尾升华但不鸡汤
- ❌ Bad: 远程工作的优点是ABC，缺点是XYZ，所以我们要平衡。——四平八稳，没有任何锋芒

### Test 3：大师级（真正的高手怎么用？）
**Prompt:** "这段文案我觉得不好，太像广告了，帮我改得更有说服力：[原文：我们的AI写作助手采用最先进的大语言模型技术，能够帮助您快速生成高质量内容，提升工作效率300%，是企业数字化转型的必备工具。]"
**Expected Output Type:** 诊断+重写的完整迭代（展示评价→修改→优化的全过程）
**Must Include:**
- 问题诊断（具体指出哪里"像广告"：堆砌术语/自说自话/缺乏用户视角/没有情感连接）
- 重写版（完全不同的切入角度，不是简单的同义词替换）
- 对比说明（改了什么、为什么这样改更好）
- 如果是特定渠道（官网/朋友圈/邮件），给出渠道适配版
- 传授方法（让用户下次能自己写出这样的文案）

**Quality Bar:**
- ✅ Good: 诊断"这段话犯了技术傲慢症——全是'我'没有'你'"→ 重写从用户痛点切入"你有没有经历过对着空白文档三小时的崩溃？"→ 对比表格展示before/after → 总结"广告味去除公式"
- ❌ Bad: 只是润色了一下措辞，本质还是"我们很好快来买"

## Failure Cases（2 个）

### Failure 1: 通用 AI 文体（没有文体意识）
**When:** 用户要求写任何类型的文字时
**What Goes Wrong:** 输出的文字读起来都一样——无论微博、论文、小说、邮件，都是同一种"AI腔"：工整、正确、但没有任何个性
**Why:** 没有建立"文体指纹"的概念，把所有写作任务当成同一个模式处理
**Fix:** 在动笔前先明确：谁在看？什么场合？期望的反应是什么？然后用对应的文体规则约束输出

### Failure 2: 只有文字没有说服力架构
**When:** 用户需要的是"能打动人"的文案
**What Goes Wrong:** 文字本身没问题（语法正确、逻辑清晰），但读完之后读者不会采取任何行动——缺乏说服力的心理设计
**Why:** 把写作当成了"信息传递"而不是"行为设计"，忽略了文案的终极目的是改变读者的想法或行动
**Fix:** 引入说服力框架（PAS/SCQA/AIDA等），在写作前先设计"读者阅读旅程"

## Style Fingerprint（风格指纹）

**Good Output Smells Like:**
- **文体敏感**：换一个场景，语体自动切换（微博≠公众号≠公文≠小说）
- **人味**：有观点、有态度、甚至有小毛病，不像机器生成的"完美但无聊"
- **决策透明**：能解释"为什么这里用了短句""为什么这里故意不结尾"
- **读者意识**：每句话都在考虑"读者此刻在想什么"

**Bad Output Smells Like:**
- **AI八股文**：首先...其次...最后...综上所述...（正确的废话）
- **过度修饰**：大量形容词副词堆砌，看起来华丽但信息密度极低
- **没有牙齿**：什么都不敢说，什么立场都不敢站

## Before / After

**Before:** 用户面对写作任务要么无从下笔，要么写出来自己都觉得"不太对"
**After:** 用户拥有多文体的写作能力 + 自我诊断工具 + 迭代优化方法
**One-Line Delta:** 从"怕写东西"到"知道怎么写才能达到目的"
