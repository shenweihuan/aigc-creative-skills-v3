# 💡 设计领域 — Design Domain

> **好的设计不是让人注意到设计本身，而是让信息自然而然地流入用户的心智。**
> **本领域 = 你的视觉秩序训练场 + AI辅助设计工作流。**

**注意：设计领域的特殊性**

```
与其他领域的区别：
- 视觉艺术 → 表达自我（"我想画什么"）
- 写作文学 → 传达思想（"我想说什么"）
- 影视视频 → 讲述故事（"我想展示什么"）
- 设计      → 解决问题（"用户需要什么"）

设计的核心不是美学，而是功能。
美学是手段，解决问题才是目的。

本领域的技能教你：
✓ 如何用大师级的品味框架做设计决策
✓ 如何用AI工具快速生成和迭代设计方案
✓ 如何评价一个设计是否"真正好用"
```

## 三级技能索引

### 🎨 平面设计 Graphic Design (`graphic/`)

**2大设计流派，覆盖现代设计史的核心范式。**

| 技能路径 | 流派/大师 | 一句话定位 | 最佳适用场景 | 关键触发词 |
|---------|----------|-----------|------------|-----------|
| `graphic/swiss-international` | 国际主义风格 / 瑞士平面设计 | 网格系统中的秩序之美 | 企业VI/画册设计/信息图表/需要高度组织化的内容；追求清晰、客观、系统化 | "国际主义""瑞士风格""网格""排版""无衬线""系统性""Helvetica" |
| `graphic/japanese-minimal` | 原研哉 / 日本极简设计（MUJI哲学） | 白的空间哲学 | 品牌设计/包装/生活方式类；追求留白、自然、无意识设计、东方禅意 | "日式极简""原研哉""白""留空""MUJI""禅意""无印良品" |

#### 平面技能对比矩阵

| 维度 | 国际主义 | 日式极简 |
|------|---------|---------|
| **核心哲学** | 秩序/网格/客观性 | 空/白/自然/无意识 |
| **视觉特征** | 非对称网格/数学比例/摄影+字体 | 大量留白/自然材质/手写感 |
| **典型色彩** | 少量原色 + 黑白灰 | 单色/大地色/纸张本色 |
| **字体选择** | 无衬线（Helvetica/Univers） | 衬线或手写体（有温度） |
| **情感温度** | 冷静/理性/专业 | 温暖/宁静/亲和 |
| **文化语境** | 欧洲现代主义/功能性优先 | 东方美学/感受性优先 |
| **适合行业** | 科技/金融/学术/政府 | 生活品牌/餐饮/文化/健康 |
| **AI生成难度** | ⭐⭐（网格结构清晰，易于描述） | ⭐⭐⭐（"留白的艺术"难以精确控制） |

### 📱 UI/UX 设计 (`ui-ux/`)

**2种界面设计哲学，从消费电子到工业经典的永恒原则。**

| 技能路径 | 流派/大师 | 一句话定位 | 最佳适用场景 | 关键触发词 |
|---------|----------|-----------|------------|-----------|
| `ui-ux/apple-design-lang` | 苹果设计语言 (Apple Design Language) | 科技与人文的十字路口 | iOS/macOS应用/消费电子产品/需要精致细节和一致性；追求简洁、直觉、愉悦感 | "苹果风""iOS""简洁""圆角""毛玻璃""现代""精致" |
| `ui-ux/dieter-rams-less` | 迪特·拉姆斯 / "少即是多"工业设计 | 功能至上的永恒原则 | 工具类应用/B端产品/工业设备界面；追求极致功能主义、去除一切多余 | "少即是多""博朗""Dieter Rams""功能主义""经典""持久" |

#### UI/UX技能对比矩阵

| 维度 | 苹果设计语言 | 拉姆斯功能主义 |
|------|------------|--------------|
| **核心哲学** | Technology + Humanity (科技+人文) | Less, but better (少，但更好) |
| **设计原则** | 直觉性/一致性/美感/效率 | 可用性/可理解性/诚实性/持久性 |
| **视觉特征** | 圆角/半透明/微妙动效/毛玻璃 | 直角/扁平/无装饰/高对比度 |
| **交互模式** | 手势驱动/流畅过渡/即时反馈 | 明确操作/最少步骤/状态可见 |
| **色彩系统** | 渐变/柔和阴影/系统蓝色调 | 黑白灰+单一强调色/无渐变 |
| **动效态度** | 动效是功能（引导注意力） | 极少或无动效（不干扰任务） |
| **典型应用** | 消费级App/游戏/创意工具 | 专业工具/企业系统/仪表盘 |
| **时代感** | 当代（2013-至今，持续演进） | 永恒（1950-70年代，至今不过时） |

**何时选择哪个？**

```
你的产品/界面是：
→ 给普通消费者用的？          → apple-design-lang（更友好/更有愉悦感）
→ 给专业人士用的工具？        → dieter-rams-less（更高效/更少干扰）
→ 需要展示"先进感"？           → apple-design-lang
→ 需要展示"可靠性"？           → dieter-rams-less
→ 目标用户年轻/追求时尚？      → apple-design-lang
→ 目标用户成熟/重视实质？      → dieter-rams-less
```

### 🔤 排版 Typography (`typography/`)

**2种极端的排版哲学，从系统化到彻底反叛。**

| 技能路径 | 大师 | 一句话定位 | 最佳适用场景 | 关键触发词 |
|---------|------|-----------|------------|-----------|
| `typography/massimo-vignelli` | 马西莫·维格内利 | 字体是设计的灵魂 | 书籍/地图/标识系统/需要高度一致性和可读性的项目；追求字体作为唯一设计元素的力量 | "维格内利""Helvetica""地铁图""系统""字体设计""经典排版" |
| `typography/david-carson-grunge` | 大卫·卡森 / 脏版运动 | 解构一切排版规则 | 音乐专辑/青年文化/实验出版/需要打破常规和制造冲击力；追求混乱中的视觉能量 | "解构""脏版""大卫·卡森""实验""反设计""Ray Gun杂志" |

#### 排版技能对比矩阵

| 维度 | 维格内利（系统） | 卡森（反叛） |
|------|---------------|-------------|
| **对规则的态度** | 规则即自由（在系统中创造变化） | 规则即限制（必须打破才有生命力） |
| **可读性** | 最高优先级（清晰传达信息） | 可被牺牲（情绪表达优先） |
| **字体使用** | 极少（1-2个字体家族，靠大小/粗细变化） | 极多（混合多种冲突字体，甚至手写/涂鸦） |
| **网格系统** | 严格遵循（但可能创造性地打破） | 故意无视（或故意错位以制造张力） |
| **空白处理** | 有意识的负空间（呼吸感） | 填满或完全随机（混乱的能量场） |
| **典型作品** | 纽约地铁图 / 联邦快递标识 / Unigrid系统 | Ray Gun杂志 / 字体档案实验海报 |
| **适用场景** | 信息设计/导视系统/品牌规范 | 演唱会海报/亚文化出版物/实验艺术 |
| **AI生成难度** | ⭐⭐（规则明确，易于描述） | ⭐⭐⭐⭐⭐（"受控的混乱"极难用AI实现） |

## 选择决策树："我需要做什么类型的设计？"

当用户说"帮我设计XX"时，使用此决策树：

```
开始：用户的设计需求是什么？
│
├─ 设计对象判断
│  ├─ "二维静态内容"              → 进入 [平面分支]
│  ├─ "数字界面/App/网页"         → 进入 [UI/UX分支]
│  ├─ "文字为主的内容"            → 进入 [排版分支]
│  └─ "不确定/混合需求"           → 进入 [通用诊断]
│
├─ [平面分支] （graphic/）
│  ├─ 需要/偏好？
│  │  ├─ 高度组织化/信息密集       → swiss-international (国际主义)
│  │  ├─ 温暖/自然/有呼吸感        → japanese-minimal (日式极简)
│  │  └─ 不确定 → 继续诊断 ↓
│  │
│  └─ 快速判断：
│     ├─ 行业是科技/金融/学术？    → swiss-international
│     ├─ 行业是生活/餐饮/文化？    → japanese-minimal
│     └─ 目标受众偏理性？          → swiss-international
│        目标受众偏感性？          → japanese-minimal
│
├─ [UI/UX分支] （ui-ux/）
│  ├─ 产品类型？
│  │  ├─ 消费级App/面向大众        → apple-design-lang (苹果)
│  │  ├─ 专业工具/B端/效率导向     → dieter-rams-less (拉姆斯)
│  │  └─ 不确定 → 继续 ↓
│  │
│  └─ 核心目标？
│     ├─ 让人"想用"/享受过程       → apple-design-lang
│     └─ 让人"高效完成任务"        → dieter-rams-less
│
├─ [排版分支] （typography/）
│  ├─ 内容性质？
│  │  ├─ 必须清晰传达信息           → massimo-vignelli (维格内利)
│  │  ├─ 想要视觉冲击/实验性        → david-carson-grunge (卡森)
│  │  └─ 不确定 → 继续 ↓
│  │
│  └─ 受众容忍度？
│     ├─ 需要快速获取信息（低容忍度）→ massimo-vignelli
│     └─ 可以花时间解读（高容忍度）  → david-carson-grunge
│
└─ [通用诊断] （5个快速问题）
   Q1: 主要功能是什么？
       → 信息传递（报告/说明书/数据）→ graphic/swiss-international 或 typography/vignelli
       → 品牌形象（logo/海报/包装）→ graphic/（根据调性选）
       → 任务完成（软件/网站/应用）→ ui-ux/
       → 情绪表达（艺术/文化活动）→ typography/carson 或 japanese-minimal

   Q2: 受众是谁？
       → 大众/泛用户 → apple-design-lang / japanese-minimal（最友好）
       → 专业人群 → dieter-rams-less / swiss-international（最高效）
       → 亚文化/青年群体 → david-carson-grunge（最有个性）

   Q3: 媒介/载体？
       → 数字屏幕（手机/电脑）→ ui-ux/（必须考虑交互）
       → 印刷品（纸质）→ graphic/ + typography/（可发挥印刷质感）
       → 环境/空间（导视/展板）→ swiss-international（导视系统最佳）

   Q4: 时间/预算约束？
       → 快速产出/低成本 → japanese-minimal 或 dieter-rams-less（元素少=快）
       → 愿意打磨/预算充足 → 任何都行（复杂度不同而已）

   Q5: 风格成熟度要求？
       → 需要"看起来专业/可信"→ swiss-international / vignelli / rams
       → 需要"看起来有趣/创新"→ carson / apple-design-lang
       → 需要"看起来有品味/高级"→ japanese-minimal / vignelli
```

## 与 AI 的协作流程（设计领域特殊说明）

> **设计领域的AI协作与视觉艺术不同：设计不是纯表达，而是问题解决。**
> **因此AI在设计流程中更多扮演"快速原型"和"方案生成"的角色，而非最终决策者。**

### 协作架构

```
用户需求："帮我们团队设计一个App的主界面"

⚠️ 重要：设计之前必须先明确问题！
│
├─ Phase 0: 问题定义（不可跳过）
│  ├── 这个App解决什么问题？（不是"做什么功能"，而是"解决什么痛点"）
│  ├── 用户是谁？他们的核心场景是什么？
│  ├── 成功标准是什么？（不是"好看"，而是具体指标）
│  └── 输出：设计简报（Design Brief），1页纸
│
├─ Phase 1: 加载设计技能 (品味层)
│  └── ui-ux/apple-design-lang/SKILL.md（假设选择苹果风格）
│      ├── 提供：苹果设计原则（直觉性/一致性/审美/效率）
│      ├── 提供：感知训练（如何识别"好的交互"）
│      ├── 提供：组件库思维（颜色/字体/间距/圆角系统）
│      └── 提供：评价标准（可用性测试清单）
│
├─ Phase 2: AI辅助概念生成 (工艺层 - ai-painting-master)
│  ├── 用AI生成多个方向的概念板（mood board）
│  │  方向A：苹果式简洁（参考apple-design-lang）
│  │  方向B：拉姆斯式功能（参考dieter-rams-less）
│  │  方向C：日式温暖（参考japanese-minimal）
│  ├── 从每个方向选最佳元素
│  └── 输出：整合后的设计方向
│
├─ Phase 3: 详细设计（人工主导 + AI辅助）
│  ├── 信息架构设计
│  ├── 页面布局（线框图）
│  ├── 视觉设计（配色/字体/图标）
│  │  ├── AI辅助生成变体（如：生成5种配色方案）
│  │  └── 人工选择和微调
│  ├── 交互设计（流程/状态/反馈）
│  └── 输出：高保真设计稿
│
├─ Phase 4: 评价与迭代
│  └── 使用对应技能的自检清单：
│      ├── apple-design-lang检查：
│      │  ✓ 是否遵循HIG（Human Interface Guidelines）？
│      │  ✓ 交互是否直觉（无需学习）？
│      │  ✓ 视觉层次是否清晰？
│      │  ✓ 动效是否有意义（非装饰性）？
│      │  ✓ 在不同设备上是否一致？
│      └── 输出：评价报告 + 修改建议
│
└─ Phase 5: 交付与规范
   ├── 切图/标注/开发对接
   ├── 设计系统文档（如果需要）
   └── 后续迭代计划
```

### 不同设计任务的AI适配策略

| 设计任务 | 推荐技能 | AI参与度 | 典型工作流 |
|---------|---------|---------|-----------|
| **Logo设计** | graphic/（任一） | 中（30%） | AI生成概念 → 人工精修 → AI变体测试 |
| **海报设计** | graphic/ + typography/ | 高（60%） | AI生成版式 → 人工调整文案/图像 → AI出多版本 |
| **UI界面** | ui-ux/ | 中（40%） | AI生成mood board → 人工做线框图 → AI辅助视觉设计 |
| **书籍/排版** | typography/vignelli | 低（20%） | 人工制定网格 → AI辅助字体配对建议 → 人工精细调整 |
| **实验性设计** | typography/carson | 高（80%） | AI大量生成 → 人工筛选/组合 → 再AI生成变体 |
| **品牌VI系统** | graphic/swiss-international | 中（30%） | AI生成基础元素 → 人工构建系统 → AI扩展应用场景 |

## 典型工作流示例

### 示例1：新手首次使用 — 设计一张海报

```
用户："帮我们社团的读书活动设计一张宣传海报"

系统响应：
  1. 问题定义（Phase 0）：
     - 目标：吸引大学生参加读书分享会
     - 关键信息：时间/地点/主题书名/主讲人
     - 基调：文艺但不老气，有吸引力

  2. 风格推荐：
     - 路由判断：[平面] → [文艺/文化] → japanese-minimal 或 swiss-international
     - 推荐：japanese-minimal（温暖/有文化气息/适合读书主题）

  3. 加载 graphic/japanese-minimal/SKILL.md
     - 学习"白的空间哲学"
     - 感知训练：分析3张MUJI海报的留白运用（10分钟）

  4. AI辅助生成（ai-painting-master）：
     - 提示词模板：
       "Minimalist book poster design, Japanese minimalist style,
       Kenya Hara philosophy, generous white space,
       single central image of open book with soft natural lighting,
        subtle texture of washi paper, warm earth tone accent color,
       clean sans-serif typography for event details at bottom,
       asymmetrical but balanced composition, elegant restraint,
       cultural sophistication without pretension"
     - 生成3-5个版本

  5. 日式极简自检：
     ✓ 留白是否足够（至少40%空白）？
     ✓ 色彩是否克制（不超过2-3色）？
     ✓ 字体是否有温度（避免过于机械的无衬线）？
     ✓ 整体是否有"呼吸感"？
     ✓ 信息层级是否清晰？

  6. 输出：最终海报设计 + 应用延展（朋友圈图/门票/现场背板）
```

### 示例2：熟练用户 — UI界面设计

```
用户："为一个时间管理工具设计iOS主界面，要苹果风格"

自动路由：
  → ui-ux/apple-design-lang/SKILL.md (品味层)

执行流程：
  1. 问题定义：
     - 核心任务：快速记录/查看日程
     - 用户场景：碎片时间使用（排队/通勤）
     - 成功指标：添加事件<3秒完成

  2. 加载苹果设计原则：
     - Intuition（直觉）：主要操作一步到达
     - Consistency（一致性）：遵循iOS规范
     - Aesthetics（审美）：简洁但不冷淡
     - Efficiency（效率）：减少点击次数

  3. AI辅助概念生成：
     - 生成3种布局方案（列表/卡片/时间轴）
     - 生成配色方案（基于iOS系统色 + 品牌色）
     - 生成图标风格参考

  4. 人工决策 + AI细化：
     - 选择时间轴布局（更适合"时间"主题）
     - 微调间距/字号（符合HIG规范）
     - AI生成不同状态的界面（空状态/加载/错误）

  5. 苹果风格自检：
     ✓ 导航是否清晰（tab bar位置正确）？
     ✓ 主要操作是否突出（CTA按钮够明显）？
     ✓ 反馈是否及时（操作后有视觉确认）？
     ✓ 是否支持Dark Mode？
     ✓ 可访问性（对比度/文字大小）？

  6. 输出：完整UI套件（5-8个关键页面）+ 交互说明 + 开发标注
```

### 示例3：跨技能融合（高级）

```
用户："设计一套独立音乐节的视觉系统，要有冲击力"

融合策略：多技能协作
  主技能：david-carson-grunge（60% - 视觉冲击/反叛精神）
  辅技能1：swiss-international（20% - 保持基本的可读性和系统化）
  辅技能2：japanese-minimal（20% - 在混乱中留出"呼吸点"）

执行方案：
  1. 主视觉（海报/主KV）：卡森全开
     - 解构字体/层叠图像/不规则构图
     - AI大量生成 → 人工暴力编辑
     - 目标：让人过目不忘

  2. 延展物料（节目单/导览图）：维格内利介入
     - 在混乱的主视觉基础上，信息部分回归有序
     - 网格系统确保时间表/地图清晰可读
     - 形成"外乱内序"的反差魅力

  3. 数字媒介（网站/App界面）：日式极简平衡
     - UI保持干净（不与主视觉抢戏）
     - 但点缀卡森式的"破坏"元素（如：一个错位的按钮）
     - 让整体既有秩序又有惊喜

  4. 最终效果预期：
     "一套看起来'几乎失控'但实际上经过精密计算的视觉系统。
      卡森提供能量，维格内利提供骨架，原研哉提供呼吸。
      这就是顶级设计师做的事——在混乱和秩序之间跳舞。"
```

### 2. 设计原则速查卡

**六大通用设计原则（适用于所有技能）：**

```
1. 对比 Contrast
   - 大小/颜色/形状/纹理的差异化
   - 目的：建立视觉层次，引导注意力

2. 重复 Repetition
   - 统一的元素贯穿全局
   - 目的：建立一致性和专业性

3. 对齐 Alignment
   - 所有元素都与某个参照对齐
   - 目的：建立秩序感和关联性

4. 亲密性 Proximity
   - 相关的元素靠近，不相关的分开
   - 目的：组织信息，减少认知负担

5. 负空间 White Space
   - 有意识地留出空白区域
   - 目的：给设计"呼吸"，突出重点

6. 层次 Hierarchy
   - 通过大小/颜色/位置区分重要性
   - 目的：引导用户的阅读顺序
```

## 设计伦理提醒

在使用AI辅助设计时，请注意：

1. **原创性**：AI生成的方案应作为起点而非终点，加入人的判断和调整
2. **版权风险**：AI可能无意中模仿现有设计，重要项目务必做原创性检查
3. **文化敏感性**：设计是全球语言，但要注意符号的文化含义差异
4. **可访问性**：不要为了"好看"牺牲可访问性（色盲友好/屏幕阅读器兼容/对比度达标）
5. **可持续性**：考虑设计的环境影响（数字碳足迹/印刷材料选择）
6. **不误导用户**：特别是UI设计，不要用设计技巧欺骗或操纵用户行为

---

> **记住：这个领域不是教你"怎么用好看的模板"，而是教你怎么"像设计师一样思考"。**
> **好看的方案有很多，但正确的方案只有一个——那个最好地解决问题的方案。**
> **AI帮你更快地探索可能性，但只有你能决定哪个可能性值得变成现实。**
