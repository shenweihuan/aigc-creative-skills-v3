# 共享模块：创作循环 (Creation Cycle)

> 统一的迭代SOP——从模糊想法到成品输出的6阶段流程。
> 适用于所有媒介（图像/文字/视频/设计），各阶段可根据需要裁剪。

## 六阶段循环

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  ① 澄清 ──→ ② 发散 ──→ ③ 收敛                    │
│  Clarify     Diverge      Converge                  │
│                                                     │
│       ↑                   │                          │
│       │                   ↓                          │
│  ⑥ 复盘 ←── ⑤ 判断 ←── ④ 深化                     │
│  Review      Decide       Deepen                    │
│                                                     │
│  （不满意则回到②或④继续迭代）                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Phase 1: 澄清 (Clarify) — "到底要做什么？"

**目标**：把模糊的想法变成精确的创作简报

**输出物**：创作简报（Project Brief）

```markdown
## 创作简报

【一句话】用一句话描述你要创作的作品
【目标受众】谁会看/读/消费这个作品？
【核心情绪】你希望观众/读者感受到什么？
【风格方向】（选定后填入对应艺术大师）
【格式约束】尺寸/时长/字数/比例/平台
【绝对不要】明确列出你不想要的东西
【参考】（如有）参考作品链接或描述
```

**时间预算**：5-10 分钟
**常见错误**：跳过这步直接开始 → 后面反复返工

---

### Phase 2: 发散 (Diverge) — "有多少种可能性？"

**目标**：大量生成候选方案，不做筛选

**方法**：

| 方法 | 操作 | 适用场景 |
|------|------|---------|
| **N×10法** | 快速生成10个完全不同的版本 | 图像/短文案 |
| **SCAMPER** | 替代/组合/调整/修改/用途其他/消除/重排 | 需要创意突破时 |
| **变量矩阵** | 列出关键变量，排列组合所有可能性 | 系列创作/品牌物料 |
| **大师视角切换** | 同一主题分别用3个不同大师的风格生成 | 不确定风格时 |

**输出物**：候选池（≥5个选项）
**时间预算**：10-20 分钟
**关键规则**：**这一阶段不评判**，数量优先于质量

---

### Phase 3: 收敛 (Converge) — "哪个最好？"

**目标**：用统一标准选出最优方案

**评分矩阵**：

| 候选 | 意图匹配(40) | 美学(20) | 可行性(20) | 独特性(20) | 总分 |
|------|------------|---------|-----------|-----------|-----|
| 方案A | | | | | |
| 方案B | | | | | |
| 方案C | | | | | |

**输出物**：选定方案 + 选择理由
**时间预算**：5-10 分钟
**关键规则**：必须有明确的拒绝理由（为什么没选其他的）

---

### Phase 4: 深化 (Deepen) — "怎么让它更好？"

**目标**：对选定方案进行精细化打磨

**方法：方向性反馈**

```
不是泛泛地说"再好一点"
而是给出精确的方向向量：

❌ "这张图再好一点"
✅ "色彩对比度增加20%，阴影部分加入更多冷色调，
   主体位置向左移动10%，背景简化减少细节"

❌ "这段文字再改改"
✅ "第二段的句子长度从平均35字降到18字以内，
   加入至少2个听觉细节，删除所有'非常'和'很'"
```

**迭代次数**：通常 2-3 轮深化即可
**时间预算**：15-30 分钟
**何时停止**：当增量改进 < 5% 时

---

### Phase 5: 判断 (Decide) — "够好了吗？"

**目标**：决定是否交付或继续迭代

**充分性测试 (Sufficiency Test)**：

```
问自己三个问题：

Q1: 如果明天就要交，这个版本能交吗？
   → YES → 准备交付
   → NO → 进入 Q2

Q2: 主要问题在哪里？（用评价框架打分）
   → 单个维度 < 3分 → 定向修复该维度（回到Phase 4）
   → 多个维度 < 3分 → 可能需要回到 Phase 2 重新发散

Q3: 再花一小时，预期提升有多大？
   → 预期提升 > 20% → 继续迭代
   → 预期提升 < 20% → 交付（边际效应递减）
```

**决策记录**：无论交付还是继续，都记录原因

---

### Phase 6: 复盘 (Review) — "学到了什么？"

**目标**：从本次创作中提取可复用的经验

**三分钟复盘模板**：

```markdown
## 本次创作复盘

【作品】一句话描述
【耗时】总用时
【迭代次数】共几轮

✅ 最成功的决策：
   （做了什么选择？为什么成功？可以复用吗？）

❌ 最大的失误：
   （哪里出了问题？根本原因是什么？下次怎么避免？）

💡 意外发现：
   （有什么意料之外的效果或洞察？）

📌 下次优化：
   （只记一条——最重要的一条改进）
```

**知识沉淀**：如果使用了 Learning System，数据自动写入 `logs/knowledge/`

---

## 循环变体

### 快速循环（日常内容，≤30分钟）

```
Clarify(2min) → Diverge(5min) → Decide(3min) → Deepen(10min) → Done
（跳过正式Converge，直觉选择 + 跳过Review，简单记录）
```

### 标准循环（精品内容，1-2小时）

```
完整6阶段，每阶段按标准时间执行
```

### 深度循环（重要项目，半天~数天）

```
在标准循环基础上：
- Phase 2 发散：生成 20+ 候选
- Phase 4 深化：5-8 轮迭代
- Phase 6 复盘：详细分析 + 写创作日志
- 循环间间隔：隔一夜再做最终判断（利用潜意识加工）
```
