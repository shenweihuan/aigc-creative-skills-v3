## 路由系统（MetaSkill Core）

> 这是整个系统的核心创新——**不是让你翻目录找技能，而是你说需求，系统自动路由到正确的技能**。

### 三级路由机制

```
Level 0: QuickStart 快速入门（新手首选，5分钟出活）
    ↓
Level 1: 意图识别（你在做什么类型的创作？）
    ↓
Level 2: 领域分类（属于哪个领域？）
    ↓
Level 3: 风格匹配（哪个大师/流派的品味最适合？）
    ↓
加载目标原子技能 → 执行完整工作流
```

### Level 0 — QuickStart 快速入门层（新手首选）

当用户是 AI 创作新手、或需求简单直接时，优先走 quickstart 路由：

| 用户意图 | quickstart 模块 | 触发条件 |
|---------|----------------|---------|
| 学写 prompt | 00-prompt-engineering | "教我写提示词" / "prompt怎么写" |
| 让 AI 画图 | 10-ai-painting | "帮我画一张" / "AI绘画" |
| 让 AI 写文案 | 30-ai-writing | "帮我写一篇" / "AI写作" |
| 做 AI 视频 | 20-ai-video | "做个视频" / "AI视频" |
| 提高 AI 效率 | 40-ai-efficiency | "AI工作流" / "AI效率工具" |

### AI大师的四大工作模式

每个 AI 大师都支持四种模式，按需切换：

## 风格诊断流程（当你不知道选什么时）

当用户说"不知道用什么风格"或描述模糊时，执行以下诊断：

### 诊断四问

```
Q1: 你的作品给谁看？（受众）
   → 大众/专业圈/自己/特定人群

Q2: 你想传达什么情绪？（情感目标）
   → 温暖/冷静/震撼/神秘/幽默/忧郁/力量

Q3: 你偏好哪种"精确度"？（风格取向）
   → A) 极度写实精确（→ 达芬奇/亚当斯/奥格威）
   → B) 有控制的模糊暗示（→ 莫奈/村上春树/王家卫）
   → C) 故意打破规则（→ 毕加索/达利/昆汀/卡森）

Q4: 你有多少时间？（约束）
   → 快速产出（→ 数字原生文案/抖音/Vlog）
   → 愿意打磨（→ 任何大师级技能都适合）
```

### 诊断结果映射

基于 Q1-Q4 的答案，推荐优先级排序的前3个技能：

| 受众×情绪×精确度 | 首选 | 备选1 | 备选2 |
|----------------|------|-------|-------|
| 大众×温暖×模糊暗示 | 莫奈印象派 | 宫崎吉卜力 | 海明威极简 |
| 专业圈×震撼×打破规则 | 毕加索立体主义 | 昆汀非线性 | 卡森解构 |
| 自己×宁静×写实精确 | 杉本博司极简 | 原研哉日式 | 塔可夫斯基诗电影 |
| 特定人群×幽默×快速 | 数字原生文案 | 抖音爆款 | Vlog叙事 |
| ... | ... | ... | ... |

## 完整工作流（从零到产出的标准路径）

### 新手路径（第一次使用）

```
Step 0  哲学奠基（20分钟）
  │ 读 shared/00-creation-philosophy.md
  │ 理解：AI创作的本质、人机契约、统一公式 C=P×E×J×I
  │ 目标：建立正确的心态——你不是在"用AI代工"，而是在"和AI共创"
  ↓
Step 1  意图表达（2分钟）
  │ 来到这里（本文件），用自然语言描述你想创作什么
  │ 例："我想画一张秋天的湖边小屋，要有温暖的感觉"
  │ 总控自动识别：[视觉] → [绘画] → [需要匹配风格]
  ↓
Step 2  风格选择（5分钟）
  │ 总控推荐2-3个最匹配的大师/风格
  │ 你可以：
  │   A) 直接接受推荐
  │   B) 说"我想试试另一个" → 浏览完整技能列表
  │   C) 说"混合一下" → 多技能融合模式（高级）
  ↓
Step 3  加载原子技能（自动）
  │ 系统加载目标大师的完整技能文件
  │ 包含：大师档案 → 感知训练 → 提示词公式 → 评价标准
  ↓
Step 4  感知热身（10分钟，可选但推荐）
  │ 跟着技能做一轮"感知训练"
  │ 不是教你技术，是训练你的眼睛/耳朵/心
  │ 例：莫奈技能会先让你看3幅《睡莲》局部，练习色彩分解
  ↓
Step 5  提示词构建（5分钟）
  │ 使用该技能专属的"提示词公式"
  │ 不是瞎填关键词，而是按公式结构化地组织意图
  ↓
Step 6  生成与迭代（15-30分钟）
  │ 生成初稿 → 用该技能的自检清单评价
  │ 不满意？进入 shared/03-creation-cycle.md 的6阶段循环
  ↓
Step 7  收获沉淀（5分钟）
  │ 记录：什么有效、什么无效、下次怎么改进
  │ 如果使用 Learning System，数据自动写入 knowledge/
  ↓
完成 ✓
```

```
选定一位大师 → 深入研究其完整技能
  → 完成全部感知训练（通常3-5个练习）
  → 用该大师风格创作5-10件作品
```

## 多技能融合模式（高级用法）

当你想要的效果不在单个大师范围内时：

### 融合方式

| 方式 | 操作 | 示例 |
|------|------|------|
| **A+B 主次融合** | 主技能70% + 辅技能30% | 莫奈色彩(主) + 毕加索构图(辅) |
| **A→B 流程借用** | 用A的感知训练 + B的提示词公式 | 用亚当斯的区域曝光思维 + 莱博维茨的戏剧光打光 |
| **A×B 跨域迁移** | 将A领域的品味迁移到B领域 | 将王家卫的情绪碎片法用于写小说 |

### 融合操作步骤

```
1. 明确主技能和辅技能
2. 分别加载两个技能文件
3. 从主技能取：核心审美哲学 + 评价标准
4. 从辅技能取：1-2个具体技法/关键词
5. 在提示词中以 "[主风格] with hints of [辅风格]" 组织
6. 评价时以主技能的标准为主，辅技能为加分项
```

## 诚实边界

### 这个系统能做到
- 为任何创作任务提供**具体可学的大师级品味框架**
- **智能路由**——不需要记住所有技能名，自然语言即可
- 每个技能包含**完整的从感知到产出的工作流**
- 支持多技能**融合创作**
- 基于 **OpenSquila MetaSkill** 规范的可组合架构

### 这个系统不能做到
- **不能替代你的审美判断**——它提供的是品味训练方法，不是品味本身
- **不能保证每次输出都是杰作**——AI生成有随机性，大师级的评价标准帮你筛选
- **不是艺术史教材**——每个大师的介绍聚焦于"对AI创作有什么用"，非学术性
- **不涵盖所有大师**——首批覆盖核心大师，持续扩展

### 最佳实践建议
1. **从一个大师开始深入**——不要贪多，先精通一个
2. **感知训练不要跳过**——这是你和只会调参数的人的区别
3. **用评价标准自我检查**——每件作品都要过一遍自检清单
4. **记录你的创作过程**——配合 Learning System 的 knowledge pipeline
5. **跨域迁移是最好的进阶**——用A学到的品味去挑战B领域
